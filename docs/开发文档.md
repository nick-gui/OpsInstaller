# 需求
当前目录是一个新的项目，这个项目主要是用于运维自动化安装各种软件使用。
场景示例：
1. 在腾讯云新采购了一台Linux系统的云主机，现在需要给这个台主机做一些定制化的配置
- 创建定制化的目录结构
- 优化系统参数
- 安装基础工具git、docker、supervisor等等
- 挂载数据磁盘
2. 这个项目需要兼容多种系统，比如centos、Ubuntu、debian等

---

# 项目开发方案

## 1. 技术选型

经过评估，采用 **Shell + Ansible** 的混合架构方案。

### 方案对比

| 对比项 | Python 方案 | Ansible 方案 |
|--------|-------------|--------------|
| 成熟度 | 需要从头构建 | 业界标准，生产验证 |
| 幂等性 | 需自行实现 | 原生支持 |
| 多系统兼容 | 需自行适配 | 完美支持主流发行版 |
| 生态 | 需自行开发 | 丰富的 roles/playbooks 生态 |
| 学习曲线 | 较高 | 运维人员易上手 |
| 批量操作 | 需自行实现 | 天然支持 |

### 选型理由

1. **Ansible 为主**：利用 Ansible 成熟的生态、幂等性设计、多系统兼容能力
2. **Shell 为辅**：处理前置检查、环境准备等简单任务
3. **轻量级**：无需额外依赖，开箱即用

## 2. 架构设计

采用 **Ansible Playbook + Roles** 的分层架构：

- **Playbooks**：定义完整的安装流程
- **Roles**：模块化的功能单元，可复用
- **Inventory**：主机清单管理
- **Variables**：灵活的配置变量

## 3. 目录设计

```
OpsInstaller/
├── ansible/                    # Ansible 核心目录
│   ├── playbooks/              # 剧本目录
│   │   ├── init-server.yaml    # 服务器初始化
│   │   ├── install-docker.yaml # Docker 安装
│   │   ├── install-tools.yaml  # 基础工具安装
│   │   └── mount-disk.yaml     # 磁盘挂载
│   ├── roles/                  # 角色目录
│   │   ├── common/             # 通用配置角色
│   │   │   ├── tasks/
│   │   │   ├── handlers/
│   │   │   ├── templates/
│   │   │   └── defaults/
│   │   ├── git/                # Git 安装角色
│   │   ├── docker/             # Docker 安装角色
│   │   ├── supervisor/         # Supervisor 安装角色
│   │   ├── sysctl/             # 系统参数优化角色
│   │   ├── directory/          # 目录结构创建角色
│   │   └── disk/               # 磁盘挂载角色
│   ├── inventory/              # 主机清单
│   │   ├── hosts               # 默认主机清单
│   │   └── hosts.example       # 示例清单
│   ├── group_vars/             # 组变量
│   │   └── all.yaml
│   ├── host_vars/              # 主机变量
│   └── ansible.cfg             # Ansible 配置文件
├── scripts/                    # Shell 辅助脚本
│   ├── pre-check.sh            # 前置环境检查
│   ├── bootstrap.sh            # 环境引导脚本
│   └── post-install.sh         # 安装后处理
├── configs/                    # 配置文件目录
│   └── examples/               # 配置示例
├── docs/                       # 文档
│   ├── 开发文档.md
│   └── 使用指南.md
├── tests/                      # 测试
├── README.md
└── INSTALL.md                  # 安装说明
```

## 4. 核心特性

1. **Ansible 原生幂等性**：重复执行不会产生副作用
2. **多系统兼容**：支持 CentOS 7+, Ubuntu 18.04+, Debian 9+
3. **模块化设计**：Roles 可独立使用和组合
4. **灵活配置**：通过变量覆盖实现定制化
5. **快速上手**：提供默认配置，开箱即用

## 5. 使用流程

1. 配置 Inventory 文件定义目标主机
2. 根据需求选择 Playbook
3. 自定义变量（可选）
4. 执行 Playbook
5. 验证安装结果
