## 快速开始

### 单机部署

1. 进入 ansible 目录

```bash
cd ansible
```

2. 执行服务器初始化

```bash
ansible-playbook playbooks/init-server.yaml
```

3. 安装 Docker

```bash
ansible-playbook playbooks/install-docker.yaml
```

### 多机部署

1. 编辑 `inventory/hosts`，添加目标服务器

2. 测试连接

```bash
ansible all -m ping
```

3. 批量执行

```bash
ansible-playbook playbooks/full-install.yaml
```

## 主机清单配置

### 基本格式

```ini
# 单台主机
[web_servers]
web01 ansible_host=192.168.1.10 ansible_user=root
web02 ansible_host=192.168.1.11 ansible_user=root

# 主机组
[all_servers:children]
web_servers
db_servers

# 组变量
[all_servers:vars]
ansible_ssh_common_args='-o StrictHostKeyChecking=no'
```

### 使用非 22 端口

```ini
web01 ansible_host=192.168.1.10 ansible_user=root ansible_port=2222
```

### 使用密码认证

```ini
web01 ansible_host=192.168.1.10 ansible_user=root ansible_password=yourpassword
```

建议使用 SSH 密钥认证，不要在配置文件中明文存储密码。

## 变量配置

### 全局变量（group_vars/all.yaml）

```yaml
# 时区
system_timezone: "Asia/Shanghai"

# 目录结构
base_directories:
  - path: "/data"
    owner: "root"
    group: "root"
    mode: "0755"
  - path: "/data/logs"
    owner: "root"
    group: "root"
    mode: "0755"

# 系统参数
sysctl_config:
  - key: "net.core.somaxconn"
    value: "65535"

# Docker 配置
docker_registry_mirrors:
  - "https://docker.mirrors.ustc.edu.cn"
```

### 主机变量（host_vars/）

为特定主机创建变量文件：

```yaml
# ansible/host_vars/web01.yaml
disk_mounts:
  - device: "/dev/vdb"
    mount_path: "/data"
    fstype: "ext4"

base_directories:
  - path: "/data"
    owner: "root"
    group: "root"
    mode: "0755"
  - path: "/data/www"
    owner: "www-data"
    group: "www-data"
    mode: "0755"
```

## Playbook 使用

### 列出所有 Playbook

```bash
ls -la ansible/playbooks/
```

### 执行 Playbook

```bash
cd ansible

# 执行完整安装
ansible-playbook playbooks/full-install.yaml

# 指定主机组
ansible-playbook playbooks/install-docker.yaml -l web_servers

# 指定单台主机
ansible-playbook playbooks/init-server.yaml -l web01
```

### 常用选项

| 选项 | 说明 |
|------|------|
| `-i inventory_file` | 指定 inventory 文件 |
| `-l subset` | 限制执行的主机 |
| `-t tags` | 只执行指定 tag 的任务 |
| `--check` |  dry-run 模式，不实际执行 |
| `--diff` | 显示文件变更差异 |
| `-v, -vv, -vvv` | 显示详细输出 |

### Dry-run 模式

先测试不实际执行：

```bash
ansible-playbook playbooks/full-install.yaml --check --diff
```

## 进阶用法

### 使用 Tag

只执行特定任务：

```bash
# 查看可用 tag
ansible-playbook playbooks/full-install.yaml --list-tags

# 只执行 Docker 相关任务
ansible-playbook playbooks/full-install.yaml -t docker
```

### 使用 --limit 限制主机

```bash
# 只在 web01 上执行
ansible-playbook playbooks/full-install.yaml --limit web01

# 在一组主机上执行
ansible-playbook playbooks/full-install.yaml --limit web_servers
```

### 使用额外变量

```bash
# 命令行传递变量
ansible-playbook playbooks/full-install.yaml -e "docker_registry_mirrors=['https://your-mirror.com']"

# 从文件加载变量
ansible-playbook playbooks/full-install.yaml -e @configs/examples/custom_vars.yaml
```

### 任务级别的变量覆盖

```yaml
- name: 安装自定义版本 Docker
  hosts: all
  roles:
    - role: docker
      vars:
        docker_version: "25.0.0"
```

## 常见问题

### 如何只更新部分配置？

使用 tag 或单独执行对应的 playbook：

```bash
# 只更新系统参数
ansible-playbook playbooks/init-server.yaml -t sysctl
```

### 如何回滚变更？

Ansible 没有直接的回滚功能，建议：
1. 先使用 `--check` 测试
2. 重要操作前备份配置
3. 编写反向操作的 playbook

### 如何查看执行记录？

```bash
# 详细输出
ansible-playbook playbooks/full-install.yaml -vvv

# 保存日志到文件
ansible-playbook playbooks/full-install.yaml | tee install.log
```
